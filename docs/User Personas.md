# 1. Primary Persona – Aarav

**Age:** 27  
**Role:** Software Developer  
**Environment:** Hybrid (Office + Remote)  
**Technical Comfort:** High  
**Productivity Tool Usage:** Minimal beyond essentials

## Behavioral Profile

- Works long, uninterrupted coding sessions
- Browser-centric workflow
- Enters deep focus states
- Frequently forgets hydration and posture
- Avoids overly complex productivity tools

## Core Pain Points

- Physical strain (neck, back, eye fatigue)
- Loss of time awareness
- Unstructured deep work sessions
- Forgetting breaks

## Goals

- Maintain focus without sacrificing health
- Build steady daily rhythm
- Avoid burnout over time

## Why Rhythm Fits

- Lightweight browser extension
- Immediate defaults
- Calm, non-intrusive reminders
- No dashboard or analytics pressure

## Primary Features Used

- Water reminder
- Posture reminder
- Eye rest reminder
- Work start/end reminders
- Focus mode

## Design Implications

- Defaults must be intelligent
- No onboarding complexity
- Clear grouped layout
- Visible counters on main screen
- Minimal text, icon-supported UI
- Schedule discipline emphasized

# Persona Strategy Summary

## Primary Anchor: Aarav

Design decisions prioritize:

- Browser-heavy workflow
- Schedule discipline
- Minimal setup
- Health + structure balance

## Secondary Focus

- Corporate environments
- Remote flexibility
- Gentle structure for new-to-productivity users

## Product Positioning Direction

Rhythm is positioned as:

> A calm, lightweight schedule and wellbeing companion built directly into the browser.

Not a productivity dashboard.  
Not a habit tracker.  
Not a task manager.
